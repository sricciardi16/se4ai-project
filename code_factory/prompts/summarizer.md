Your task is to write a concise summary of a source code file.

The purpose of this summary is triage. Given a future project modification or a problem to solve, a reader must be able to read your summary and immediately decide: "Do I need to open this file, or can I discard it?"

**Requirements:**

- **Focus:** Highlight the specific responsibilities, behaviors, and logic that this file owns within the project.
- **Brevity:** Keep it strictly between 3 to 5 sentences.
- **Abstraction:** Focus on the "What" and the "Why". Do not explain how the code works line-by-line.
- **Format:**  Place your summary under the markdown header: ### Summary
