Project: `web_mapper`


## 1. High-Level Goal
Implement a Python library named `web_mapper` that generates interactive web maps. The library must provide an object-oriented API where users can instantiate a map, attach various geographic layers and UI controls to it, and lazily compile the entire object tree into a standalone HTML/JavaScript document.

## 2. Module Structure
You must create the following module structure:
*   `web_mapper`: The root module containing the core classes.
*   `web_mapper.plugins`: A submodule containing specialized plugin classes.

## 3. Core Architecture & Lazy Rendering
*   **Unique Identifiers:** Every map element (Layers, Markers, Controls) must generate and store a unique string identifier upon instantiation.
*   **Lazy Evaluation:** Elements are attached to the map via an `add_to(map_instance)` method. The actual HTML/JS generation must **not** happen during `add_to()`. It must happen lazily when the map's rendering methods (`render()`, `save()`, or `_repr_html_()`) are explicitly called.

## 4. Classes & Functions

### 4.1. `Map`
The central class representing the web map.

*   **Signature:** `Map(location=None, zoom_start=None, tiles=None, attr=None)`
*   **Attributes:**
    *   `location`: A public property storing the map's center coordinates as a list of two floats `[latitude, longitude]`.
*   **Behaviors & Rules:**
    *   **Location Initialization:** If `location` is provided as a sequence of floats, store it exactly as provided. If `location` is `None` or omitted, default the `location` attribute to exactly `[0.0, 0.0]`.
    *   **Tileset Validation:** Maintain an internal registry of recognized built-in tileset names (e.g., `'CartoDB dark_matter'`). If the `tiles` argument is provided and is *not* in this recognized list, treat it as a custom URL.
    *   **Attribution Enforcement:** If `tiles` is treated as a custom URL, you MUST check the `attr` argument. If `attr` is `None` or omitted, raise a `ValueError` with the exact message: `"Custom tiles must have an attribution"`.
*   **Methods:**
    *   `get_root()`: Returns an object representing the root of the DOM/HTML tree. This root object must have a `render()` method that returns the final compiled HTML as a string.
    *   `save(filepath)`: Compiles the map into a standalone HTML document and writes it to the specified `filepath` using UTF-8 encoding. The output file must not be empty, must start with `<!DOCTYPE html>`, and must contain `<html` and `<div class="`.
    *   `_repr_html_()`: A dunder method for Jupyter Notebook integration. It must return a string containing either an `<iframe` or `<div` tag, and must include either `srcdoc=` or `data-html=` attributes containing the inline HTML/JS payload.
    *   **Rendering Requirements:** The final HTML output generated by the map must include references to the Leaflet JavaScript library (the output string must contain `"leaflet.js"` or `"leaflet"` case-insensitively).

### 4.2. `Marker`
A class representing a point of interest on the map.

*   **Signature:** `Marker(location, tooltip=None, popup=None, icon=None, name=None)`
*   **Methods:**
    *   `add_to(map_instance)`: Attaches the marker to the map.
    *   `get_name()`: Returns the marker's unique string identifier.
*   **Behaviors & Rules:**
    *   **Coordinate Validation:** Validate that the `location` argument contains numeric values (integers or floats). If they are strings or non-numeric, raise a `ValueError`. Do **not** validate geographic bounds (e.g., `[999.999, -999.999]` must be accepted without error).
    *   **Rendering:** When the map is rendered, the exact string representations of the latitude and longitude must be embedded in the HTML output. The HTML must also contain the marker's unique name.
    *   **Popup:** If `popup` is provided (which may contain complex HTML markup, quotes, and `<script>` tags), the core text and payload of the popup must be embedded in the final HTML output (either raw, HTML-escaped, or JSON-escaped).
    *   **Tooltip:** If `tooltip` is provided (which may contain non-ASCII Unicode characters), the exact string must be embedded natively in the HTML output.

### 4.3. `CircleMarker`
A class representing a circular point on the map.

*   **Signature:** `CircleMarker(location, radius)`
*   **Methods:** `add_to(map_instance)`, `get_name()`
*   **Behaviors & Rules:** When added to a map, its unique name must be present in the rendered HTML output.

### 4.4. `Icon`
A class representing a custom visual icon for a Marker.

*   **Signature:** `Icon(color=None, icon=None, prefix=None)`
*   **Behaviors & Rules:** When passed to a `Marker` via the `icon` argument, the specific strings provided for `color`, `icon`, and `prefix` must be embedded as CSS classes or configuration properties in the rendered HTML output.

### 4.5. `TileLayer`
A class representing a map tile layer.

*   **Signature:** `TileLayer(tiles, name=None, attr=None)`
*   **Methods:** `add_to(map_instance)`, `get_name()`
*   **Behaviors & Rules:**
    *   **Attribution Enforcement:** Exactly like the `Map` class, if `tiles` is an unrecognized built-in string, treat it as a custom URL. If `attr` is `None` or omitted, raise a `ValueError` with the exact message: `"Custom tiles must have an attribution"`.
    *   **Rendering:** When added to a map, its unique name and its custom `name` argument (if provided) must be present in the rendered HTML output.

### 4.6. `LayerControl`
A UI element that allows users to toggle layers on and off.

*   **Signature:** `LayerControl()`
*   **Methods:** `add_to(map_instance)`, `get_name()`
*   **Behaviors & Rules:**
    *   **Lazy Layer Discovery:** The LayerControl must dynamically discover all layers attached to the map at the time the map is *rendered*, **not** at the time the LayerControl is instantiated or added. If a layer is added to the map *after* the LayerControl is added, the LayerControl must still include that layer's `name` in the final HTML output.
    *   **Rendering:** Its unique name must be present in the HTML output.

### 4.7. `GeoJson`
A class for rendering GeoJSON vector data.

*   **Signature:** `GeoJson(data, style_function=None, name=None)`
*   **Methods:** `add_to(map_instance)`
*   **Behaviors & Rules:**
    *   **Data Serialization:** The `data` argument (a Python dictionary representing GeoJSON) must be serialized and embedded into the HTML output. The output must contain the geometry types (e.g., `"FeatureCollection"`, `"Point"`, `"LineString"`, `"Polygon"`) and the exact coordinate arrays provided in the dictionary.
    *   **Style Function:** If `style_function` is provided, it will be a callable that accepts a GeoJSON feature dictionary and returns a dictionary of CSS properties (e.g., `{"color": "red", "weight": 2}`). The library must apply this function to the features and serialize the resulting CSS properties into the HTML output (e.g., `'"color": "red"'`).

### 4.8. `Choropleth`
A class for creating color-mapped regions based on statistical data.

*   **Signature:** `Choropleth(geo_data, data, columns, key_on, fill_color=None)`
*   **Methods:** `add_to(map_instance)`
*   **Behaviors & Rules:**
    *   **Data Binding:** `data` will be a `pandas.DataFrame`. `columns` will be a list of two strings representing the columns to use.
    *   **Column Validation:** You must verify that the strings provided in `columns` exist in the `data` DataFrame. If they do not exist, raise a `KeyError`.
    *   **Color Validation:** Maintain a predefined set of valid color scales/palettes. If `fill_color` is provided and is not recognized, raise a `ValueError`.
    *   **Missing Data Handling:** If the geographic identifiers in `geo_data` (mapped via `key_on`) do not have matching keys in the `data` DataFrame, the map must still render successfully without raising an exception. The unmatched regions must be assigned a default missing-data color, and the rendered HTML must contain `"black"` or `"#000000"`.
    *   **Rendering:** The geographic identifiers from `geo_data` must be present in the rendered HTML output.

### 4.9. `web_mapper.plugins.MarkerCluster`
A plugin class for clustering multiple markers.

*   **Signature:** `MarkerCluster(name=None)`
*   **Methods:** `add_to(map_instance)`, `get_name()`
*   **Behaviors & Rules:**
    *   **Empty State:** The cluster must render successfully even if zero child markers are added to it.
    *   **Rendering:** When added to a map, the rendered HTML output must include the cluster's unique name, as well as the JavaScript asset references `"markerClusterGroup"` and `"markercluster"` (case-insensitive).
